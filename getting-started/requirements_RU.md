# Требования

Перед настройкой WakeLink убедитесь, что у вас есть необходимое оборудование и программное обеспечение.

## Требования к оборудованию

### ESP-плата (обязательно)

Вам понадобится микроконтроллер ESP8266 или ESP32. Это недорогие платы с поддержкой WiFi, которые будут отправлять Wake-on-LAN пакеты.

| Плата | Цена | Примечания |
|-------|------|-----------|
| **ESP8266 (NodeMCU)** | ~$3–5 | Рекомендуется. Широко доступна, надёжна |
| **ESP32** | ~$5–8 | Мощнее, поддержка Bluetooth (для WakeLink не нужна) |
| **Wemos D1 Mini** | ~$3–4 | Компактная форм-фактор, на базе ESP8266 |

!!! tip "Где купить"
    - Amazon, AliExpress, eBay
    - Местные магазины электроники
    - Ищите «NodeMCU ESP8266» или «ESP32 DevKit»

### USB-кабель

- Micro-USB кабель (для большинства плат ESP8266)
- USB-C кабель (для более новых плат ESP32)

Используется только для первоначальной прошивки. После настройки ESP работает самостоятельно.

### Целевой компьютер

Ваш компьютер должен поддерживать Wake-on-LAN:

- [x] Ethernet-порт (WiFi WoL ненадёжен)
- [x] Сетевая карта с поддержкой WoL (большинство современных NIC)
- [x] Подключённый к сети блок питания (компьютер должен быть включён в розетку)

!!! warning "Ограничения WoL"
    - Работает из состояния **выключения** или **сна**
    - **НЕ** работает при принудительном обесточивании
    - Должен быть включён в BIOS и операционной системе

### Требования к сети

- WiFi-сеть (2.4 ГГц для ESP8266)
- ESP и целевой компьютер должны быть в **одной локальной сети**
- Интернет-соединение (для relay-взаимодействия)

## Программные требования

### Для прошивки

=== "Arduino IDE"
    - [Arduino IDE 2.x](https://www.arduino.cc/en/software)
    - Поддержка плат ESP8266/ESP32
    - Необходимые библиотеки (устанавливаются автоматически)

=== "PlatformIO"
    - [VS Code](https://code.visualstudio.com/)
    - [Расширение PlatformIO](https://platformio.org/install/ide?install=vscode)

=== "Веб-прошивальщик"
    - Современный браузер (Chrome, Edge, Firefox)
    - Установка не требуется

### Для сервера (собственный хостинг)

- [Docker](https://docs.docker.com/get-docker/) и Docker Compose
- Минимум 512 МБ ОЗУ
- Рекомендуется Linux (Windows/macOS тоже работает)

### Для клиентов

=== "CLI"
    - Python 3.9+
    - pip (менеджер пакетов Python)

=== "Android"
    - Android 8.0+ (API 26)
    - Google Play Store или установка APK вручную

## Настройка Wake-on-LAN на компьютере

### 1. Включение в BIOS/UEFI

1. Войдите в BIOS (обычно Del, F2 или F12 при загрузке)
2. Найдите раздел Power Management или Network settings
3. Включите **Wake on LAN** / **Wake on PCI** / **Power On By PCI-E**
4. Сохраните и выйдите

!!! note "Названия в BIOS различаются"
    Ищите: «Wake on LAN», «WoL», «Power On by Network», «Wake on PCI/PCI-E»

### 2. Включение в операционной системе

=== "Windows 11/10"
    1. Откройте **Диспетчер устройств**
    2. Разверните **Сетевые адаптеры**
    3. Щёлкните правой кнопкой по Ethernet-адаптеру → **Свойства**
    4. Перейдите на вкладку **Управление электропитанием**
    5. Установите флажки:
        - ✅ Разрешить этому устройству выводить компьютер из ждущего режима
        - ✅ Разрешить только магическому пакету выводить компьютер из ждущего режима
    6. Перейдите на вкладку **Дополнительно**
    7. Установите **Wake on Magic Packet** = Включено

=== "Windows (Отключение быстрого запуска)"
    Быстрый запуск может мешать работе WoL:

    1. Откройте **Панель управления** → **Электропитание**
    2. Нажмите **Действия кнопок питания**
    3. Нажмите **Изменение параметров, которые сейчас недоступны**
    4. Снимите флажок **Включить быстрый запуск**
    5. Сохраните изменения

=== "Linux"
    ```bash
    # Установите ethtool
    sudo apt install ethtool

    # Проверьте текущий статус WoL
    sudo ethtool eth0 | grep Wake

    # Включите WoL (g = magic packet)
    sudo ethtool -s eth0 wol g
    ```

    Сделайте постоянным (systemd):
    ```bash
    # /etc/systemd/system/wol.service
    [Unit]
    Description=Enable Wake-on-LAN
    After=network.target

    [Service]
    Type=oneshot
    ExecStart=/sbin/ethtool -s eth0 wol g

    [Install]
    WantedBy=multi-user.target
    ```

=== "macOS"
    1. **Системные настройки** → **Экономия энергии** (или Аккумулятор)
    2. Установите флажок **Выход из режима сна для сетевого доступа**

### 3. Узнайте MAC-адрес

Для настройки вам понадобится MAC-адрес целевого компьютера.

=== "Windows"
    ```cmd
    ipconfig /all
    ```
    Найдите «Физический адрес» в разделе вашего Ethernet-адаптера.

=== "Linux"
    ```bash
    ip link show
    # или
    cat /sys/class/net/eth0/address
    ```

=== "macOS"
    ```bash
    ifconfig en0 | grep ether
    ```

!!! example "Формат MAC-адреса"
    ```
    AA:BB:CC:DD:EE:FF
    ```
    WakeLink принимает форматы: `AA:BB:CC:DD:EE:FF`, `AA-BB-CC-DD-EE-FF`, `AABBCCDDEEFF`
