# Введение

WakeLink обеспечивает безопасное пробуждение по сети (Wake-on-LAN) через Интернет с использованием сквозного шифрования между вашими устройствами.

## Зачем нужен WakeLink?

Традиционный Wake-on-LAN (WoL) имеет фундаментальное ограничение: магические пакеты работают только в локальной сети. Чтобы разбудить домашний компьютер, находясь в поездке, как правило требуется:

- Проброс портов на роутере (угроза безопасности)
- Настройка VPN (сложно, требует постоянного подключения)
- Сторонние сервисы (часто закрытый исходный код, проблемы с конфиденциальностью)

**WakeLink решает эту задачу** простым, безопасным и открытым способом:

| Проблема | Решение WakeLink |
|----------|-----------------|
| Магические пакеты работают только локально | Relay-сервер на WebSocket соединяет устройства |
| Проблемы безопасности | Сквозное шифрование (XChaCha20-Poly1305) |
| Настройка роутера | Проброс портов не требуется |
| Конфиденциальность | Самостоятельный хостинг, открытый исходный код |
| Сложность | Настройка за 5 минут |

## Ключевые возможности

### 🔐 Сквозное шифрование

Всё взаимодействие между клиентом и ESP-устройством шифруется. Relay-сервер **не может читать ваши команды** — он лишь пересылает зашифрованные пакеты.

- **Алгоритм**: XChaCha20-Poly1305 (AEAD)
- **Обмен ключами**: Pre-shared key + HKDF деривация
- **Защита от повторных атак**: Порядковые номера + цепочечное хэширование

### 🌐 Работает из любой точки мира

Будите компьютер из офиса, из поездки или из другой страны. Достаточно наличия доступа в Интернет.

### 📡 Несколько платформ

- **ESP8266/ESP32**: Недорогие устройства с низким потреблением (~$3–5)
- **CLI**: Полнофункциональный инструмент командной строки
- **Android**: Нативное мобильное приложение
- **API**: REST-эндпоинты для интеграции

### 🏠 Самостоятельный хостинг

Запустите собственный relay-сервер с помощью Docker. Полный контроль над данными и инфраструктурой.

## Обзор архитектуры

```
┌─────────────┐          ┌─────────────┐          ┌─────────────┐
│   Client    │◄────────►│   Server    │◄────────►│ ESP Device  │
│  (CLI/App)  │   HTTPS  │   (Relay)   │    WSS   │  (Firmware) │
└─────────────┘          └─────────────┘          └──────┬──────┘
                                                         │
                                                    WoL Packet
                                                         │
                                                         ▼
                                                  ┌─────────────┐
                                                  │  Target PC  │
                                                  └─────────────┘
```

**Поток взаимодействия**:

1. Клиент аутентифицируется на сервере
2. Клиент отправляет зашифрованную команду пробуждения
3. Сервер пересылает её на ESP-устройство через постоянное WebSocket-соединение
4. ESP расшифровывает и проверяет подпись команды
5. ESP отправляет магический WoL-пакет в локальную сеть
6. Целевой компьютер просыпается
7. ESP отправляет зашифрованное подтверждение

## Протокол: EWSP v1.0

WakeLink использует **Encrypted Wake Signaling Protocol (EWSP)** — специализированный протокол, разработанный с учётом:

- **Конфиденциальности**: Все данные зашифрованы
- **Целостности**: HMAC-проверка каждого пакета
- **Актуальности**: Защита от повторных атак через цепочечное хэширование
- **Эффективности**: Минимальные накладные расходы для ресурсоограниченных устройств
