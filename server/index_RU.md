# Обзор сервера

Разверните собственный ретранслятор WakeLink для полного контроля над своими данными.

## Зачем использовать самостоятельный хостинг?

| Преимущество | Описание |
|--------------|----------|
| **Конфиденциальность** | Ваши данные остаются на вашей инфраструктуре |
| **Контроль** | Настраивайте всё под себя |
| **Надёжность** | Нет зависимости от сторонних сервисов |
| **Без ограничений** | Неограниченное количество устройств и запросов |
| **Свой домен** | Используйте собственный домен и брендинг |

## Компоненты

| Сервис | Назначение | Порт |
|--------|------------|------|
| **API Server** | REST API, управление устройствами | 8000 |
| **WebSocket Gateway** | Подключения устройств | 8001 |
| **PostgreSQL** | Данные пользователей/устройств | 5432 |
| **Redis** | Сессии, pub/sub | 6379 |
| **Nginx** | Обратный прокси, TLS | 80/443 |

## Требования

### Железо (минимум)

| Ресурс | Требование |
|--------|------------|
| CPU | 1 ядро |
| RAM | 1 ГБ |
| Хранилище | 10 ГБ |
| Сеть | 100 Мбит/с |

| Ресурс | Требование |
|--------|------------|
| CPU | 2+ ядра |
| RAM | 4 ГБ |
| Хранилище | 50 ГБ SSD |
| Сеть | 1 Гбит/с |

### Программное обеспечение

- Docker 20.10+
- Docker Compose 2.0+
- Linux (рекомендуется Ubuntu 22.04)

## Быстрый старт

```bash
# Clone repository
git clone https://github.com/wakelinkdev/wakelink-server
cd wakelink-server

# Configure
cp .env.example .env
# Edit .env with your settings

# Start
docker-compose up -d

# Check status
docker-compose ps
```

Сервер доступен по адресу `http://localhost:8000`. Подробнее см. [Руководство по Docker](docker.md).

## Конечные точки API

| Конечная точка | Описание |
|----------------|----------|
| `POST /api/v1/auth/login` | Аутентификация пользователя |

| `GET /metrics` | Метрики Prometheus |

## Вопросы безопасности

1. **Всегда используйте TLS** — Никогда не открывайте сервер без HTTPS
2. **Надёжные секреты** — Генерируйте случайный JWT-секрет и пароль БД
3. **Файрвол** — Открывайте только порты 80/443
4. **Обновления** — Поддерживайте актуальность Docker-образов
5. **Резервные копии** — Регулярно выполняйте резервное копирование базы данных

Подробнее см. [Лучшие практики безопасности](../security/best-practices.md).
