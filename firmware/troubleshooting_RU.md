# Устранение неполадок

Распространённые проблемы и решения для прошивки WakeLink.

## Диагностические инструменты

### Serial Monitor

Первый шаг при любом устранении неполадок:

```bash
# PlatformIO
pio device monitor -b 115200

# Arduino IDE
Tools → Serial Monitor (115200 baud)
```

### Команда статуса

```bash
wakelink status my-device
```

Вывод содержит:
- Состояние подключения
- Временную метку последней активности
- Версию прошивки
- Уровень сигнала WiFi

## Проблемы с WiFi

### Устройство не подключается к WiFi

**Симптомы:**
- Медленное мигание светодиода
- Serial: `[WIFI] Connecting...` повторяется

**Решения:**

1. **Проверьте учётные данные**
   ```
   config set wifi.ssid "ExactNetworkName"
   config set wifi.password "ExactPassword"
   config save
   ```

2. **Проверьте диапазон 2,4 ГГц** — ESP8266 не поддерживает 5 ГГц

3. **Проверьте специальные символы** — некоторые роутеры имеют проблемы с пробелами или символами в SSID

4. **Переместитесь ближе к роутеру** — слабый сигнал вызывает сбои подключения

5. **Проверьте фильтрацию MAC-адресов** — добавьте MAC-адрес ESP в белый список на роутере

### WiFi подключён, но нет соединения с сервером

**Симптомы:**
- Быстрое мигание светодиода (WiFi OK, сервер недоступен)
- Serial: `[WS] Connection failed`

**Решения:**

1. **Проверьте URL сервера**
   ```
   config get server.url
   # Должно быть wss://relay.wakelink.io или ваш сервер
   ```

2. **Проверьте брандмауэр** — исходящий порт 443 (HTTPS/WSS) должен быть открыт

3. **Разрешение DNS** — попробуйте IP-адрес вместо имени хоста

4. **Проверьте TLS** — собственные серверы требуют действительных сертификатов

### WiFi постоянно отключается

**Симптомы:**
- Частые переподключения в логах
- Нестабильная доступность

**Решения:**

1. **Блок питания** — используйте стабильный источник 5 В, избегайте USB-портов ПК
2. **Помехи WiFi** — уберите от микроволновок и Bluetooth-устройств
3. **Перегрузка роутера** — слишком много подключённых устройств
4. **Обновление прошивки** — в старых версиях были проблемы со стабильностью

## Проблемы с Wake-on-LAN

### Команда пробуждения отправлена, но ПК не просыпается

Это наиболее распространённая проблема. Проверяйте по порядку:

#### 1. Проверьте MAC-адрес

```bash
# Проверить настроенный MAC
wakelink config my-device | grep mac

# Сравнить с фактическим MAC ПК (на самом ПК)
ipconfig /all  # Windows
ip link show   # Linux
```

#### 2. Проверьте включение WoL на ПК

**Настройки BIOS:**
- Wake on LAN: Enabled
- Wake on PCI-E: Enabled
- ErP/EuP: Disabled (экономит энергию, но блокирует WoL)

**Windows:**
1. Диспетчер устройств → Сетевые адаптеры → Ваш адаптер → Свойства
2. Управление питанием: ✅ Разрешить этому устройству выводить компьютер из ждущего режима
3. Дополнительно: Wake on Magic Packet = Enabled

**Отключите быстрый запуск (Windows):**
```
Панель управления → Электропитание → Действия кнопок питания
→ Изменить параметры → Снять галочку «Включить быстрый запуск»
```

#### 3. Тест в локальной сети

Отправьте WoL с другого устройства в той же сети:

```bash
# Linux
wakeonlan AA:BB:CC:DD:EE:FF

# Windows (со сторонним инструментом)
wolcmd AABBCCDDEEFF 192.168.1.255 255.255.255.0 9

# Python
pip install wakeonlan
wakeonlan AA:BB:CC:DD:EE:FF
```

Если локальный WoL работает, а WakeLink — нет, проблема на стороне ESP-устройства.

#### 4. Проверьте сеть ESP

ESP должен находиться в **той же подсети**, что и целевой ПК:

```
IP ESP:      192.168.1.100
IP ПК:       192.168.1.50
             └── Одна подсеть ✅

IP ESP:      192.168.1.100
IP ПК:       192.168.2.50
             └── Разные подсети ❌
```

#### 5. Проверьте широковещательный адрес

По умолчанию `255.255.255.255`. Некоторые сети требуют широковещательный адрес подсети:

```json
{
  "target": {
    "mac": "AA:BB:CC:DD:EE:FF",
    "broadcast": "192.168.1.255"
  }
}
```

## Проблемы с подключением к серверу

### «Authentication Failed» (ошибка аутентификации)

**Симптомы:**
- Serial: `[AUTH] Token rejected`
- CLI: `Error: Device not authenticated`

**Решения:**

1. **Сгенерируйте токен заново**
   ```bash
   wakelink token regenerate my-device
   ```
   
2. **Обновите конфигурацию устройства** с новым токеном

3. **Проверьте срок действия токена** — токены могут истечь после длительного бездействия

### «Device Offline» (устройство не в сети)

**Симптомы:**
- CLI: `Error: Device offline`
- На панели управления отображается статус offline

**Решения:**

1. **Проверьте физическое устройство** — оно включено?
2. **Проверьте WiFi** — можете ли вы пинговать устройство?
3. **Проверьте serial-логи** — есть ли сообщения об ошибках?
4. **Перезапустите устройство** — иногда WebSocket требует переподключения

### «Connection Timeout» (таймаут соединения)

**Симптомы:**
- Команды зависают, затем истекает таймаут
- Serial: `[WS] Ping timeout`

**Решения:**

1. **Проверьте интернет** — устройству нужно стабильное соединение
2. **Проверьте статус сервера** — посетите status.wakelink.io
3. **Уменьшите интервал ping** — сеть может быть нестабильной
   ```json
   {
     "server": {
       "ping_interval_ms": 15000
     }
   }
   ```

## Проблемы с инициализацией

### Не найдена сеть «WakeLink-Setup»

**Решения:**

1. **Подождите 30 секунд** — точке доступа нужно время для запуска
2. **Сброс к заводским настройкам** — удерживайте кнопку FLASH 10+ секунд
3. **Проверьте, не настроено ли уже устройство** — возможно, оно пытается подключиться к сохранённому WiFi
4. **ESP может быть повреждён** — попробуйте другую плату

### Captive Portal не открывается

**Решения:**

1. **Перейдите вручную** — откройте http://192.168.4.1
2. **Отключите мобильные данные** — телефон может предпочесть 4G
3. **Попробуйте другое устройство** — некоторые реализации captive portal имеют проблемы
4. **Очистите кэш браузера**

### Конфигурация сохраняется, но не применяется

**Решения:**

1. **Дождитесь перезагрузки** — устройство должно перезагрузиться после сохранения
2. **Проверьте serial** — ищите ошибки валидации конфигурации
3. **Сброс к заводским настройкам и повторите**

## Справочник поведения светодиода

| Паттерн | Значение | Действие |
|---------|----------|----------|
| Выключен | Нет питания или глубокий сон | Проверьте блок питания |
| Горит постоянно | Подключено и готово | Нормальная работа |
| Медленное мигание (1/сек) | Подключение к WiFi | Подождите или проверьте учётные данные |
| Быстрое мигание (4/сек) | WiFi OK, подключение к серверу | Проверьте URL/токен сервера |
| Двойное мигание | Получена команда | Нормально |
| Тройное мигание | Произошла ошибка | Проверьте serial-логи |
| Непрерывное быстрое мигание | Идёт обновление прошивки | Не отключайте питание! |

## Проблемы с памятью

### Ошибки «Out of Memory»

**Симптомы:**
- Устройство падает/перезагружается
- Serial: `[HEAP] Low memory warning`

**Решения:**

1. **Уменьшите буфер JSON** — файл конфигурации слишком большой
2. **Отключите режим отладки** — отладочное логирование использует RAM
3. **Обновите прошивку** — утечки памяти могут быть исправлены
4. **Используйте ESP32** — больше оперативной памяти

### Конфигурация теряется после перезагрузки

**Решения:**

1. **Проверьте состояние Flash** — Flash может быть изношен
2. **Убедитесь в успешности сохранения** — serial должен показывать `[CONFIG] Saved`
3. **Проверьте стабильность питания** — провалы напряжения могут повредить Flash

## Процедуры восстановления

### Сброс к заводским настройкам

**Метод 1: Кнопка**
1. Выключите устройство
2. Удерживайте кнопку FLASH
3. Включите, продолжая удерживать
4. Держите 10 секунд
5. Отпустите — устройство входит в режим инициализации

**Метод 2: Serial**
```
config reset
```

**Метод 3: Стирание Flash**
```bash
esptool.py --port COM3 erase_flash
```
Затем повторно прошейте прошивку.

### Восстановление после неудачного OTA-обновления

Если устройство не загружается после OTA:

1. **Подождите** — сторожевой таймер должен запустить откат (30 секунд)
2. **Если откат не произошёл** — сброс к заводским настройкам через кнопку
3. **Если кнопка не работает** — прошейте заново через USB

### Неработающее устройство

Если устройство кажется полностью нереагирующим:

1. **Проверьте питание** — светодиод вообще горит?
2. **Попробуйте другой USB-кабель**
3. **Попробуйте другой USB-порт**
4. **Прошейте в режиме загрузки**:
   - Удержите FLASH + RST
   - Отпустите RST
   - Отпустите FLASH
   - Устройство теперь в режиме загрузки
