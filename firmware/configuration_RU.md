# Настройка прошивки

Настройка параметров устройства WakeLink.

## Первоначальная настройка (режим инициализации)

Если у устройства нет сохранённой конфигурации, оно запускается в **режиме инициализации**:

1. Подключитесь к WiFi-сети: `WakeLink-Setup`
2. Откройте [http://192.168.4.1](http://192.168.4.1)
3. Заполните форму настройки
4. Нажмите **Save**

!!! info "Captive Portal"
    На большинстве устройств страница настройки открывается автоматически при подключении к сети настройки.

## Параметры конфигурации

### Основные настройки

| Параметр | Описание | Обязательный |
|----------|----------|--------------|
| **Имя устройства** | Понятное имя (например, `office-pc`) | Да |
| **WiFi SSID** | Название вашей WiFi-сети | Да |
| **Пароль WiFi** | Пароль вашей WiFi-сети | Да |
| **URL сервера** | Адрес WebSocket-сервера | Да |
| **Токен устройства** | Токен аутентификации с сервера | Да |
| **Целевой MAC** | MAC-адрес ПК для пробуждения | Да |

### Расширенные настройки

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| **Порт сервера** | Порт WebSocket | 443 |
| **Использовать TLS** | Включить зашифрованное соединение | true |
| **Локальный режим** | Включить локальный HTTP-сервер | false |
| **Локальный порт** | Порт для локального HTTP-сервера | 80 |
| **Глубокий сон** | Включить энергосбережение | false |
| **Интервал сна** | Минуты между проверками | 5 |
| **Светодиод включён** | Индикаторы состояния LED | true |
| **OTA включён** | Разрешить удалённое обновление прошивки | true |
| **Режим отладки** | Подробное логирование через serial | false |

## Методы конфигурации

### Метод 1: Веб-интерфейс

Веб-интерфейс инициализации — самый простой метод.

**Доступ к интерфейсу:**

1. **Первый раз**: устройство автоматически запускается в режиме инициализации
2. **Перенастройка**: удерживайте кнопку FLASH 5 секунд при загрузке
3. **Сброс к заводским настройкам**: удерживайте кнопку FLASH 10 секунд

### Метод 2: Serial-консоль

Настройка через serial-команды (115200 бод):

```bash
# Просмотр текущей конфигурации
config show

# Установка отдельных значений
config set wifi.ssid "MyNetwork"
config set wifi.password "MyPassword"
config set server.url "wss://relay.wakelink.io"
config set device.token "abc123..."
config set target.mac "AA:BB:CC:DD:EE:FF"
config set device.name "office-pc"

# Сохранение и применение
config save
config apply

# Сброс к заводским настройкам
config reset
```

### Метод 3: JSON-файл

Загрузите файл `config.json` через веб-интерфейс:

```json
{
  "device": {
    "name": "office-pc",
    "token": "your-device-token-here"
  },
  "wifi": {
    "ssid": "MyNetwork",
    "password": "MyPassword"
  },
  "server": {
    "url": "wss://relay.wakelink.io",
    "port": 443,
    "tls": true
  },
  "target": {
    "mac": "AA:BB:CC:DD:EE:FF",
    "broadcast": "255.255.255.255",
    "port": 9
  },
  "local": {
    "enabled": false,
    "port": 80,
    "auth_required": true
  },
  "power": {
    "deep_sleep": false,
    "sleep_interval_minutes": 5
  },
  "features": {
    "led_enabled": true,
    "ota_enabled": true,
    "debug_mode": false
  }
}
```

## Форматы URL сервера

| Формат | Пример |
|--------|--------|
| Публичный ретранслятор | `wss://relay.wakelink.io` |
| Собственный сервер (TLS) | `wss://your-server.com:443` |
| Собственный сервер (без TLS) | `ws://your-server.local:8080` |
| IP-адрес | `wss://203.0.113.50:443` |

!!! warning "Безопасность"
    Всегда используйте `wss://` (WebSocket Secure) для продуктивных развёртываний.

## Целевой MAC-адрес

MAC-адрес компьютера, который вы хотите разбудить.

### Допустимые форматы

Все следующие форматы допустимы:

```
AA:BB:CC:DD:EE:FF
AA-BB-CC-DD-EE-FF
AABBCCDDEEFF
aa:bb:cc:dd:ee:ff
```

### Как узнать MAC-адрес

Смотрите [Руководство по требованиям](../getting-started/requirements.md#3-find-your-mac-address)

---

## Несколько целевых адресов

Вы можете настроить несколько MAC-адресов для группового пробуждения:

```json
{
  "targets": [
    {
      "name": "main-pc",
      "mac": "AA:BB:CC:DD:EE:FF"
    },
    {
      "name": "backup-server",
      "mac": "11:22:33:44:55:66"
    }
  ]
}
```

Затем разбудите конкретный адрес:

```bash
wakelink wake office-pc --target main-pc
wakelink wake office-pc --target backup-server
wakelink wake office-pc --all  # Разбудить все адреса
```

## Локальный режим

Включите локальный HTTP-сервер для работы только в LAN:

```json
{
  "local": {
    "enabled": true,
    "port": 80,
    "auth_required": true,
    "auth_token": "local-secret-token"
  }
}
```

### Конечные точки локального API

| Конечная точка | Метод | Описание |
|----------------|-------|----------|
| `GET /` | GET | Страница статуса |
| `GET /status` | GET | Статус в формате JSON |
| `POST /wake` | POST | Отправить команду пробуждения |
| `GET /config` | GET | Текущая конфигурация |
| `POST /config` | POST | Обновить конфигурацию |

### Локальный запрос пробуждения

```bash
curl -X POST http://192.168.1.100/wake \
  -H "Authorization: Bearer local-secret-token" \
  -H "Content-Type: application/json" \
  -d '{"target": "main-pc"}'
```

## Управление питанием

### Режим глубокого сна

Для развёртываний с питанием от батареи:

```json
{
  "power": {
    "deep_sleep": true,
    "sleep_interval_minutes": 5,
    "wake_on_schedule": true,
    "schedule": ["08:00", "12:00", "18:00"]
  }
}
```

**Принцип работы:**

1. Устройство просыпается из глубокого сна
2. Подключается к WiFi и серверу
3. Проверяет ожидающие команды (окно 30 секунд)
4. Выполняет все ожидающие команды
5. Возвращается в глубокий сон

!!! warning "Ограничение глубокого сна"
    В режиме глубокого сна команды пробуждения не мгновенны — устройство проверяет их только с заданным интервалом.

## Конфигурация GPIO

Настройте пины светодиода и кнопки:

```json
{
  "gpio": {
    "led_pin": 2,
    "led_inverted": true,
    "button_pin": 0,
    "button_inverted": true
  }
}
```

| Плата | Светодиод по умолчанию | Кнопка по умолчанию |
|-------|------------------------|---------------------|
| NodeMCU | GPIO2 (инверсный) | GPIO0 |
| Wemos D1 Mini | GPIO2 (инверсный) | GPIO0 |
| ESP32 DevKit | GPIO2 | GPIO0 |

## Валидация

Устройство проверяет конфигурацию перед применением:

| Проверка | Ошибка |
|----------|--------|
| Длина WiFi SSID | 1–32 символа |
| Пароль WiFi | 0 или 8–63 символа |
| Формат MAC-адреса | Корректный hex, 6 байт |
| URL сервера | Корректный URL ws/wss |
| Токен | Непустая строка |

Некорректная конфигурация отображает ошибку в веб-интерфейсе и serial-консоли.

---

## Сброс к заводским настройкам

Три способа сброса:

1. **Кнопка**: удерживайте FLASH 10+ секунд при загрузке
2. **Serial**: команда `config reset`
3. **Веб-интерфейс**: кнопка сброса на странице инициализации

Сброс удаляет все настройки и перезапускает устройство в режиме инициализации.

## Резервное копирование и восстановление

### Экспорт конфигурации

```bash
# Через serial
config export

# Через веб-интерфейс
GET http://192.168.4.1/config/export
```

### Импорт конфигурации

```bash
# Через serial (вставьте JSON)
config import {"device":{"name":"test"},...}

# Через веб-интерфейс
POST http://192.168.4.1/config/import
Content-Type: application/json
{...configuration JSON...}
```
