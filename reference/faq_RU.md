# FAQ

Часто задаваемые вопросы о WakeLink.

## Общее

### Что такое WakeLink?

WakeLink — это система с открытым исходным кодом для удалённого пробуждения компьютеров через Интернет с использованием магических пакетов Wake-on-LAN (WoL) и сквозным шифрованием для обеспечения безопасности.

### Чем WakeLink отличается от обычного Wake-on-LAN?

Обычный WoL работает только в локальных сетях. WakeLink добавляет:
- Подключение через Интернет с помощью релейного сервера
- Сквозное шифрование
- Отсутствие необходимости пробрасывать порты
- Мобильное приложение и CLI

### WakeLink бесплатный?

Да, WakeLink полностью бесплатный и с открытым исходным кодом (лицензия MIT). Вы также можете самостоятельно развернуть его для полного контроля.

### Нужны ли технические знания?

Базовая настройка требует минимальных технических знаний. Расширенные функции (самостоятельный хостинг, кастомная конфигурация) требуют большей экспертизы.

## Железо

### Какие платы ESP поддерживаются?

- ESP8266 (NodeMCU, Wemos D1 Mini и др.)
- ESP32 (все варианты, включая S2, C3)

ESP8266 рекомендуется за простоту и низкую стоимость.

### Можно ли использовать Raspberry Pi?

Пока нет, но вклад сообщества приветствуется. Протокол задокументирован для портирования.

### Сколько стоит железо?

Платы ESP8266 обычно стоят 3–5 USD. Это единственная необходимая покупка.

### ESP должен быть всегда включён?

Да, ESP должен оставаться под питанием для получения команд пробуждения. Потребление тока приблизительно:
- Активный режим: ~70 мА
- Глубокий сон: ~20 мкА (с периодическим пробуждением)

## Настройка

### ESP не подключается к WiFi

1. Убедитесь в правильности SSID и пароля
2. Убедитесь, что сеть работает на частоте 2,4 ГГц (ESP8266 не поддерживает 5 ГГц)
3. Проверьте наличие спецсимволов в учётных данных
4. Переместитесь ближе к роутеру

Подробнее см. [Устранение неполадок](../firmware/troubleshooting.md).

### Мой компьютер не просыпается

1. **Включите WoL в BIOS**: найдите "Wake on LAN" или "Wake on PCI-E"
2. **Включите в ОС**: Диспетчер устройств → Сетевой адаптер → Управление электропитанием
3. **Отключите быстрый запуск** (Windows)
4. **Проверьте MAC-адрес**: проверьте опечатки
5. **Используйте Ethernet**: WoL по WiFi ненадёжен

См. [Требования](../getting-started/requirements.md#pc-wake-on-lan-setup).

### Как узнать MAC-адрес компьютера?

**Windows**: `ipconfig /all` → найдите "Physical Address"
**Linux**: `ip link show` или `cat /sys/class/net/eth0/address`
**macOS**: `ifconfig en0 | grep ether`

### Можно ли будить несколько компьютеров?

Да, настройте несколько целей на одном устройстве ESP или используйте несколько устройств ESP.

## Безопасность

### WakeLink безопасен?

Да. WakeLink использует:
- Шифрование XChaCha20-Poly1305 (того же класса, что TLS 1.3)
- Сквозное шифрование (сервер не может читать команды)
- HMAC-аутентификацию
- Защиту от повторного воспроизведения

См. [Обзор безопасности](../security/index.md).

### Может ли сервер видеть мои команды пробуждения?

Нет. Команды зашифрованы сквозным шифрованием. Сервер видит только зашифрованные блоки данных.

### Что делать, если кто-то получит мой токен устройства?

Он сможет отправлять команды пробуждения на ваше устройство. При компрометации:
```bash
wakelink token regenerate my-device
```
Затем обновите токен на ESP.

### Стоит ли разворачивать сервер самостоятельно?

Самостоятельный хостинг обеспечивает максимальный контроль, но требует обслуживания. Публичный сервер безопасен благодаря сквозному шифрованию — мы всё равно не можем читать ваши команды.

## Подключение

### Нужно ли открывать порты на роутере?

Нет. WakeLink использует только исходящие WebSocket-соединения.

### Работает ли WakeLink за CGNAT?

Да, поскольку используются исходящие соединения.

### Что произойдёт, если ESP потеряет WiFi?

Устройство автоматически переподключится. Команды пробуждения, отправленные в период офлайн, будут поставлены в очередь для доставки.

### Какова задержка?

Типичная сквозная задержка составляет 100–500 мс в зависимости от условий сети.

## Сервер

### Можно ли развернуть сервер самостоятельно?

Да! См. [Развёртывание сервера](../server/docker.md).

### Каковы требования к серверу?

Минимум: 1 CPU, 1 ГБ ОЗУ, 10 ГБ хранилища. Работает на любом VPS или домашнем сервере.

### Есть ли хостинговый вариант?

Да, публичный релей по адресу `relay.wakelink.io` доступен бесплатно.

### Сколько устройств может обслуживать один сервер?

Один сервер может обслуживать более 10 000 устройств. Для большего числа доступно горизонтальное масштабирование.

## Мобильное приложение и CLI

### Есть ли приложение для iOS?

Пока нет, но оно запланировано. API доступен для сторонних интеграций.

### Можно ли использовать WakeLink из скрипта?

Да, CLI разработан для использования в скриптах:
```bash
wakelink wake my-pc
```

### Как автоматизировать команды пробуждения?

Используйте cron (Linux/macOS) или Планировщик задач (Windows):
```bash
# Будить компьютер в 8:00 по будням
0 8 * * 1-5 wakelink wake office-pc
```

## Устранение неполадок

### Ошибка "Device offline"

1. Проверьте питание ESP
2. Проверьте WiFi на ESP (состояние LED)
3. Проверьте соединение с сервером (логи с последовательного порта)
4. Перезапустите ESP

### "Authentication failed"

1. Убедитесь в правильности токена
2. Войдите снова в CLI: `wakelink login`
3. При необходимости пересоздайте токен устройства

### Ошибка "Timeout"

1. Устройство может быть не в сети
2. Проблемы с сетью
3. Увеличьте тайм-аут: `wakelink wake my-device --timeout 60`

### LED мигает, но компьютер не просыпается

1. WoL не включён на компьютере
2. Неверный MAC-адрес
3. Компьютер не в той же подсети, что и ESP
4. Включён быстрый запуск (Windows)

## Обновления

### Как обновить прошивку?

По воздуху (OTA):
```bash
wakelink ota update my-device
```

Или перепрошейте через USB с помощью Web Flasher.

### Как обновить сервер?

```bash
docker-compose pull
docker-compose up -d
```

### Как обновить CLI?

```bash
pip install --upgrade wakelink-cli
```
