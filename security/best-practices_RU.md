# Лучшие практики безопасности

Рекомендации по безопасному развёртыванию WakeLink.

## Безопасность устройства

### Физическая безопасность

1. **Надёжное расположение**: разместите ESP-устройство в защищённом месте
2. **Закрытые контакты**: закройте разъёмы UART/отладки
3. **Обнаружение взлома**: рассмотрите корпус с датчиком вскрытия

### Безопасность прошивки

1. **Шифрование флэш-памяти** (ESP32):
   ```bash
   # Enable in menuconfig
   idf.py menuconfig
   # Security features → Enable flash encryption
   ```

2. **Безопасная загрузка** (ESP32):
   ```bash
   # Security features → Enable secure boot
   ```

3. **Отключите отладочный вывод** в продакшене:
   ```cpp
   // config.h
   #define DEBUG_MODE false
   ```

### Безопасность токенов

1. **Никогда не публикуйте токены устройств** публично
2. **Ротируйте токены** при подозрении на компрометацию:
   ```bash
   wakelink token regenerate my-device
   ```
3. **Используйте уникальные токены** для каждого устройства

## Безопасность сервера

### Конфигурация

```bash
# .env - Security settings
DEBUG=false
LOG_LEVEL=WARNING

# Strong, unique secret
JWT_SECRET=$(openssl rand -hex 32)

# Restrict CORS
CORS_ORIGINS=https://your-domain.com

# Enable rate limiting
RATE_LIMIT_ENABLED=true
RATE_LIMIT_PER_MINUTE=60
```

### База данных

1. **Надёжный пароль**:
   ```bash
   DB_PASSWORD=$(openssl rand -hex 32)
   ```

2. **Сетевая изоляция**:
   ```yaml
   # docker-compose.yml
   services:
     db:
       networks:
         - internal  # Not exposed to host
   ```

3. **Регулярные резервные копии** с шифрованием

### Конфигурация TLS

1. **Только TLS 1.3** (если клиенты поддерживают):
   ```nginx
   ssl_protocols TLSv1.3;
   ```

2. **Надёжные шифры**:
   ```nginx
   ssl_ciphers ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384;
   ```

3. **Заголовок HSTS**:
   ```nginx
   add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
   ```

### Брандмауэр

```bash
# Only allow necessary ports
ufw default deny incoming
ufw default allow outgoing
ufw allow 22/tcp    # SSH (restrict to your IP)
ufw allow 80/tcp    # HTTP (for ACME)
ufw allow 443/tcp   # HTTPS
ufw enable
```

## Сетевая безопасность

### Домашняя сеть

1. **Отдельная IoT-сеть**: разместите ESP на изолированном VLAN
2. **Без UPnP**: отключите UPnP на маршрутизаторе
3. **Без переброса портов**: WakeLink в этом не нуждается

```
┌─────────────────────────────────────────┐
│              Home Network                │
│  ┌──────────────┐  ┌──────────────────┐ │
│  │  Main VLAN   │  │   IoT VLAN       │ │
│  │  (Trusted)   │  │   (Isolated)     │ │
│  │              │  │                  │ │
│  │  PC, Laptop  │  │  ESP, Cameras    │ │
│  └──────────────┘  └──────────────────┘ │
│          │                  │           │
│          └────────┬─────────┘           │
│                   │                     │
│            ┌──────▼──────┐              │
│            │   Firewall   │              │
│            │  (Block IoT→Main)          │
│            └─────────────┘              │
└─────────────────────────────────────────┘
```

### Безопасность DNS

1. Используйте **DNS-over-HTTPS** или **DNS-over-TLS**
2. Рассмотрите **Pi-hole** для локального DNS с фильтрацией

## Безопасность аккаунта

### Надёжные пароли

- Минимум 12 символов
- Сочетание букв, цифр и символов
- Используйте менеджер паролей

### Двухфакторная аутентификация

Включите 2FA при наличии (запланированная функция).

### Управление сессиями

1. Выходите из неиспользуемых сессий
2. Периодически проверяйте активные сессии
3. Отзывайте подозрительные сессии

## Мониторинг и оповещения

### Настройка оповещений

```yaml
# prometheus/alerts.yml
- alert: HighErrorRate
  expr: rate(http_requests_total{status=~"4.."}[5m]) > 0.1
  annotations:
    summary: "Possible attack - high 4xx error rate"

- alert: UnusualActivity
  expr: rate(wakelink_wake_commands_total[1h]) > 100
  annotations:
    summary: "Unusual wake command volume"
```

### Анализ логов

1. **Регулярный анализ логов**: проверяйте на аномалии
2. **Оповещения о неудачных входах**: отслеживайте ошибки аутентификации
3. **Геолокационные оповещения**: уведомляйте о доступе из новых местоположений

## Реагирование на инциденты

### При компрометации токена

1. **Немедленно** перегенерируйте токен:
   ```bash
   wakelink token regenerate <device>
   ```
2. Обновите ESP-устройство с новым токеном
3. Проверьте логи на предмет несанкционированного доступа
4. Рассмотрите сброс устройства к заводским настройкам

### При компрометации сервера

1. Отключите сервер
2. Ротируйте все секреты (JWT, пароль БД)
3. Принудительно завершите все сессии пользователей
4. Перегенерируйте все токены устройств
5. Проверьте журналы доступа
6. Восстановитесь из чистой резервной копии

### При компрометации устройства

1. Удалите устройство из аккаунта
2. Сбросьте устройство к заводским настройкам
3. Повторно зарегистрируйте с новым токеном
4. Проверьте сеть на предмет горизонтального распространения угрозы

## Вопросы соответствия требованиям

### GDPR

- Данные пользователей можно экспортировать (`wakelink export`)
- Данные пользователей можно удалить (`wakelink delete-account`)
- Избыточный сбор данных не ведётся

### Хранение данных

- Логи хранятся 30 дней (настраивается)
- Сессионные данные истекают через 24 часа
- История включения по желанию
